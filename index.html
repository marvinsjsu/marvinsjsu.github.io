kicksmix


Dedicated to a subculture of shoes. KicksMix is a social site that promotes the expression of ideas and conversation through kicks.

Summary

KicksMix is my final project for App Academy. My intention for this project was to keep the interface simple and focus on implementing as many back-end features I could within 11 days. It starts with the user login and session management. This allows a user to register and login to the site, otherwise, the collection of shoes comments, and mixes is on display, yet adding comments and mixes is only available once the user logs in.

During the implementation of user, shoe, and comment models, I realized these models need to be Polymorphic, as I wanted each object to be "commentable". This helped me avoid creating additional tables to keep track of comments for each of my models. In addition, as my comment model is "commentable" it worked very well for having infinitely-nested comments, where I implemented the recursion via jBuilder partials.

The client-side is implemented with Backbone. We learned the CompositeView pattern in class, so it's on this project in varying degrees. What I mean is that some elements are implemented as subviews as I wanted to see for myself the pros and cons of how deep to go about in doing subviews. This helped to learn more about event bubbling.

There are still many items on my to-do list for this project; so it is a work-in-progress as I try to fix some bugs, add some more features like tagging, and clean-up the CSS and displays. Also, for client-side session/user management, I implemented bootstrapping technique, and will need to be replaced with a more secure implementation.

asteroids


It's a nod to the classic arcade game, Asteroids. A ship finds itself in the middle of Asteroids ready to be wrecked. The player must avoid and/or shoot the asteroids down, but watch out, some of the asteroids grow and can explode and destroy your ship.

Summary

This was an awesome project as it's the first game I've written for a browser. Being able to explore and play with HTML5's Canvas and write the components in Object-Oriented Javascript is what prompted me to focus on Javascript as the language I have to get down and dirty with. In this project, OOP Javascript was implemented utilizing techniques for prototypal inheritance using a "surrogate" class and also namespacing, following the Module pattern.
